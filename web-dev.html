SAML DotNet Connector

Developed for a client of the organization, it was one of the most challenging and enriching assignments of my professional career. The objective was to design and develop an ASP.NET Connector for single sign-on to various ASP.NET applications. End-to-end development process involved designing, coding, testing, enhancement, uploading and publishing to various sites. IdentityProviders (IdP) like miniOrange, Google, ADFS etc. authenticate the credentials entered by the user to above ASP.NET applications and provide single sign-on login to these applications, all at once. The Connector provides a user interface, which takes user credentials as input, passes it onto the IdP for authentication and reads the SAML** request and response. ASP.NET applications cannot read SAML messages. However, by using this connector these applications appear to read and respond to SAML and consequently, act as Service Providers (SP) for the user.

**SAML (Security Assertion Markup Language) is a protocol, which provides a common language between SPs and IdPs and facilitate communication between the two.
